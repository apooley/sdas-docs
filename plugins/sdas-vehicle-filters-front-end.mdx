## What visitors see when Vehicle Filters is working

SDAS Vehicle Filters turns your inventory page into a searchable experience where shoppers narrow results without full page reloads.

Visitors typically see:

- A **filter area** with controls like make, model, year, price, body style, and condition.
- A **results grid** of matching vehicles that updates instantly when filters change.
- A **URL that updates** as they filter, so they can bookmark or share a link and return to the same results later.
- **Back/forward buttons** that step through previous filter settings instead of taking them to a different page.

<Callout kind="info">

SDAS Vehicle Filters depends on SDAS Inventory. Activate and configure SDAS Inventory first so the filters have an inventory source to query.

</Callout>

At a high level, you place shortcodes on a WordPress page, and the plugin takes care of:

- Reading filter choices from the page.
- Sending those choices to SDAS Inventory.
- Updating the results grid via AJAX.
- Keeping the browser URL and tracking IDs in sync.

## Core building blocks: the two shortcodes

You build your inventory search page with two shortcodes:

- **`[sdas_filters]`** — shows the filter panel that visitors interact with.
- **`[sdas_filter_results]`** — shows the vehicle results grid that updates when filters change.

### `[sdas_filters]` — the filter panel

`[sdas_filters]` renders the filter controls visitors use to narrow inventory. The plugin decides which filter types to show based on your inventory configuration.

**Available Attributes:**

| Attribute | Default | Description |
| :--- | :--- | :--- |
| `filter_layout` | `classic` | Layout style for the filter panel. Options: `classic`, `topbar_chips`, `stacked_compact`, `sidebar_desktop_modal_mobile`. The legacy `layout` attribute (`horizontal` or `vertical`) is now an alias for `classic`. |
| `filter_types` | `make,model,year_range,price_range,body_type,condition,transmission,seats` | Comma-separated list of filters to display. |
| `filter_order` | (empty) | Comma-separated list to explicitly override the default display order of filters. If not provided, filters are ordered by internal priority: `make`, `price_range`, `body_type`, `condition`, `model`, `year_range`, `mileage_range`, `transmission`, `fuel_type`, `seats`, `status`, `location`, `colour`, `style`, `drivetrain`, `keyword`. |
| `show_counts` | `yes` | Whether to show the number of matching vehicles next to each filter option. |
| `show_reset` | `yes` | Whether to display the "Clear Filters" button. |
| `show_quick_presets` | `yes` | Whether to display quick preset buttons for common filter combinations. |
| `show_vehicle_search` | `no` | Whether to display an inline vehicle search input field within the filter panel. |
| `target_selector` | `.sdas-vf-results-container` | CSS selector for the results container that the filters will update. `auto` will attempt to detect the vehicle loop automatically. |
| `auto_detect` | `yes` | Enable automatic detection of vehicle loops for page builders like Breakdance or Beaver Builder. If `target_selector` is explicitly set, `auto_detect` will be ignored. |

Example:
```text
[sdas_filters filter_layout="sidebar_desktop_modal_mobile" filter_types="make,model,price_range" show_counts="no" show_quick_presets="yes"]
```

### `[sdas_filter_results]` — the results grid

`[sdas_filter_results]` renders the actual inventory cards and pagination that respond to filter changes.

**Available Attributes:**

| Attribute | Default | Description |
| :--- | :--- | :--- |
| `per_page` | `12` | Number of vehicles to display per page. |
| `pagination` | `numbers` | Pagination style: `numbers`, `simple`, or `load_more`. |
| `card_style` | `default` | Visual style for the vehicle cards. |

Example:
```text
[sdas_filter_results per_page="24" pagination="load_more"]
```

## Advanced Front-end Features

### Built-in Sorting
The results grid includes a built-in sort dropdown that allows visitors to reorder vehicles by:
- Date (Newest/Oldest)
- Price (Low to High / High to Low)
- Mileage (Low to High / High to Low)
- Year (Newest/Oldest)

### URL and History Management
The plugin automatically manages the browser history. When a visitor applies a filter:
1. The URL is updated with query parameters (e.g., `?make=honda&model=civic`).
2. A new history state is pushed, allowing the **Back** button to return to the previous filter state.
3. Tracking identifiers (like `gclid` or `fbclid`) are preserved across filter changes and stored in cookies for lead attribution.

### Page Builder Compatibility
SDAS Vehicle Filters is designed to work seamlessly with page builders like **Breakdance**, **Beaver Builder**, and **Elementor**. The `auto_detect="yes"` setting allows the filters to find and update vehicle loops rendered by these builders without manual CSS selector configuration.

## Setup: create an inventory-with-filters page

<Steps>

<Step title="Confirm prerequisites" icon="check-circle">
Ensure SDAS Inventory and SDAS Vehicle Filters are active.
</Step>

<Step title="Create your search page" icon="edit">
Create a new page (e.g., "Search Inventory").
</Step>

<Step title="Add the shortcodes" icon="settings">
Add `[sdas_filters]` and `[sdas_filter_results]` to the page. For a sidebar layout, use a two-column block and place the filters in the narrow column with `filter_layout="sidebar_desktop_modal_mobile"`.
</Step>

<Step title="Test and Verify" icon="play-circle">
Verify that filters update the results without a page reload and that the URL updates accordingly.
</Step>

</Steps>
