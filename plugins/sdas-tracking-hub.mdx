## Overview

SDAS Tracking Hub provides a centralized system for managing and dispatching tracking events from your WordPress site to various analytics and advertising platforms. It captures front-end events, processes them server-side, and forwards them to services like Google Analytics 4 (GA4), Google Ads, and Meta (Facebook) with robust queuing and retry mechanisms. This ensures reliable data delivery for accurate reporting and campaign optimization.

## Admin Navigation and Interface

All SDAS Tracking Hub functionalities are accessible through the centralized **SDAS** menu in the WordPress admin sidebar. Navigate to **SDAS -> Tracking Hub** to access the main configuration and monitoring interface.

The Tracking Hub interface is organized into four distinct tabs, each serving a specific purpose:

-   **Overview**: Provides a high-level summary of the Tracking Hub's status, including any critical alerts or warnings regarding its operation. This is your first stop for quickly assessing the health of your tracking setup.
-   **Settings**: Configure the core behavior of the Tracking Hub. This includes setting up API credentials for external platforms (Google Ads, Meta, GA4), defining log levels, and managing other global tracking options. Sensitive credentials are securely stored using SDAS Core's token storage.
-   **Event Config**: This is where you define and manage the specific tracking events that the hub will process. You can configure how events are mapped to external platforms, set up conditions for when they fire, and ensure all necessary data fields are captured. The system provides validation to highlight events missing required fields or with incorrect configurations.
-   **Logs**: Displays a detailed log of all tracking events processed by the hub, including their delivery status to external platforms. This tab is invaluable for debugging, verifying event delivery, and troubleshooting any issues that may arise.

### Admin Notices and Troubleshooting

The Tracking Hub actively displays admin notices to inform you of important runtime prerequisites and potential issues. These notices can highlight scenarios such as:

-   **Feature Flag Disabled**: The Tracking Hub's core functionality might be turned off.
-   **WPForms Missing**: If lead tracking is enabled, but WPForms (or another integrated form plugin) is not active.
-   **Action Scheduler Missing**: The background processing system required for event queuing and retries is not available.
-   **REST Endpoint Unavailable**: The `/sdas/v1/track` endpoint, which receives front-end events, is not accessible.
-   **No Events Configured / All Events Disabled**: No tracking events are set up or enabled, meaning no data will be sent.
-   **Invalid Events**: Specific events are misconfigured, preventing their proper processing.

Always review these notices to ensure your tracking setup is operating correctly.

## Server-Side Tracking Endpoint: `POST /wp-json/sdas/v1/track`

The Tracking Hub exposes a dedicated REST API endpoint for receiving tracking events from your front-end. This server-side approach enhances data reliability and security compared to client-side-only tracking.

-   **Method**: `POST`
-   **Route**: `/sdas/v1/track`
-   **Purpose**: To receive JSON payloads containing tracking event data from your website's front-end or other services.

Front-end implementations (e.g., custom JavaScript or the SDAS Tracking Hub front-end plugin) send event data to this endpoint. The hub then processes this data, normalizes it, and dispatches it to the configured external analytics and advertising platforms.

## External Integrations

The SDAS Tracking Hub supports integration with several key external platforms:

-   **Google Analytics 4 (GA4)**: Sends event data to your GA4 property for comprehensive website analytics.
-   **Google Ads**: Facilitates conversion tracking for your Google Ads campaigns, allowing for better ad performance measurement and optimization.
-   **Meta (Facebook) Conversions API**: Enables server-side event sending to Meta, improving the accuracy of your Facebook and Instagram ad targeting and measurement, especially in privacy-centric environments.

Configuration for these integrations, including API keys and specific event mappings, is managed within the **Settings** and **Event Config** tabs of the Tracking Hub admin interface.
