## Overview

SDAS Forms Extension enhances lead capture within WordPress by integrating with supported form plugins, primarily WPForms. It processes form submissions, normalizes lead data, and provides a dedicated admin interface for managing leads, configuring form mappings, and setting up ADF (Auto-lead Data Format) emails. This plugin is crucial for dealerships to efficiently capture, manage, and synchronize customer inquiries.

## Admin Navigation and Interface

All SDAS Forms Extension functionalities are accessible through the centralized **SDAS** menu in the WordPress admin sidebar. Within the SDAS menu, navigate to **Forms** to access the main configuration area, and **Leads** to manage submitted leads.

### SDAS -> Forms

This section provides a tabbed interface for configuring how forms integrate with SDAS, defining lead types, and setting up ADF emails. The available tabs are:

-   **Form Mapping**: Configure how fields from your WPForms map to the standardized SDAS lead schema. This ensures consistency in lead data regardless of the source form.
-   **System Mapping**: Define mappings for system-level and tracking fields (e.g., UTM parameters, GCLID, referrer URL) that are automatically captured with each lead. This tab helps in attributing lead sources and understanding user behavior.
-   **Lead Types**: Manage custom lead types. This allows you to categorize leads based on their origin or intent, which can be useful for routing and reporting.
-   **ADF Emails**: Set up and configure ADF (Auto-lead Data Format) emails. This feature is essential for sending standardized lead data to external systems like CRMs or other dealership management systems.

### SDAS -> Leads

This is the central inbox for all leads captured through SDAS Forms Extension. It provides a comprehensive view of submitted leads and tools for managing them.

-   **Lead Views**: Filter leads by `All Leads`, `My Leads` (assigned to the current user), or `Unassigned` leads. This helps in organizing and prioritizing lead follow-up.
-   **Filters**: Utilize various filters such as `status`, `type`, `assignment`, and `priority` to quickly find specific leads.
-   **Pagination**: Leads are paginated, typically showing 50 leads per page, to ensure efficient browsing of large datasets.
-   **CRM Sync Status**: When the SDAS CRM Connector is active, this view displays the synchronization status of leads with external CRM systems, indicating whether a lead has been successfully synced, is pending, or has failed.
-   **Operational Actions**: Perform actions on individual leads or in bulk, such as updating lead status, assigning leads, or triggering a CRM resync for failed attempts.

## Lead Processing Workflow

SDAS Forms Extension integrates with WPForms to capture lead data. When a WPForms submission is completed, the plugin processes the data, normalizes it into the SDAS lead format, and then triggers the `sdas_lead_submitted` action. This action serves as a central integration point for other SDAS plugins, such as the CRM Connector, to pick up and process the new lead.
