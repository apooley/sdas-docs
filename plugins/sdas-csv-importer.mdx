## Overview

The SDAS CSV Importer is a robust tool designed to automate the ingestion of vehicle inventory data from external CSV feeds. It handles the entire lifecycle of an import, from fetching the remote file and parsing its contents to managing background processing and optimizing vehicle images for the web.

## Admin Interface and Configuration

The importer is managed through two primary screens located under the **SDAS** menu in the WordPress admin.

### Import Settings

The **Import Settings** screen (`SDAS -> Import Settings`) is where you configure the core behavior of the inventory synchronization:

-   **Inventory Feed**: Provide the URL to your remote CSV inventory feed. You can also define a list of **Allowed Feed Domains** to enhance security.
-   **Import Schedule**: Choose how frequently the importer should check for updates (e.g., Hourly, Twice Daily, Daily).
-   **Missing Vehicle Action**: Define what happens to vehicles that are present in your WordPress site but missing from the latest CSV feed. Options include:
    -   **Do Nothing**: Keep the vehicle as is.
    -   **Mark as Draft**: Unpublish the vehicle but keep the data.
    -   **Delete**: Permanently remove the vehicle from the site.
-   **Performance Tuning**: Adjust the **Chunk Size** (number of vehicles processed per background batch) to balance import speed with server stability.
-   **Notifications**: Set a **Notification Email** to receive alerts if an import fails or encounters significant errors.

### Image Handling and Optimization

The importer includes sophisticated image processing capabilities, configurable via the **Images** tab in settings:

-   **Storage Strategy**: Choose whether to **Download images to the Media Library** (better for SEO and reliability) or store them as **External URLs** (faster initial import).
-   **Stock Image Filtering**: Automatically exclude generic stock images from your galleries by detecting "stock_image" patterns in URLs.
-   **WPCompress Integration**: Enable CDN-based image optimization, allowing you to serve compressed, WebP-formatted images directly from the WPCompress network.
-   **Re-Image Integration**: Connect to the Re-Image API to automatically process and enhance vehicle photos during the import process.

## The Import Process

The importer operates as a multi-stage pipeline designed for reliability and performance.

1.  **Fetch & Parse**: The system retrieves the CSV file from the configured URL and parses it into structured data.
2.  **Queueing**: Instead of processing everything at once, the importer adds vehicles to a background queue.
3.  **Background Processing**: Using the WordPress Cron system and Action Scheduler, the importer processes the queue in small "chunks." This prevents server timeouts and ensures the site remains responsive for visitors.
4.  **Data Mapping**: CSV columns are automatically mapped to the SDAS Inventory schema, including complex attributes like mechanical specs, pricing, and features.
5.  **Image Pipeline**: As vehicles are imported, their images are sent to the image processing pipeline (WPCompress or Re-Image) for optimization.
6.  **Cleanup**: After the import completes, the system performs a cleanup of temporary data and executes the "Missing Vehicle Action" if configured.

## Monitoring and Troubleshooting

### Import Logs

The **Import Logs** screen (`SDAS -> Import Logs`) provides a detailed history of all synchronization attempts. You can monitor:

-   **Status**: See at a glance if an import was successful, completed with warnings, or failed.
-   **Details**: Drill down into specific log entries to see which vehicles were added, updated, or skipped.
-   **Log Levels**: Adjust the verbosity of logging (Error, Warning, Info, Debug) in settings to help diagnose complex issues.

### Common Troubleshooting Steps

-   **Verify Cron**: Ensure that WordPress Cron is functioning correctly, as it is required to drive the background import batches.
-   **Check Feed Access**: Confirm that the web server can reach the remote CSV URL and that the domain is in the "Allowed Feed Domains" list.
-   **Monitor Memory**: If imports are failing on large files, try reducing the **Chunk Size** in settings.
