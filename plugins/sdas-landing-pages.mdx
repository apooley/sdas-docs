## Overview

The SDAS Landing Pages system allows dealerships to create highly targeted, SEO-optimized pages for specific makes, models, body types, or locations. These pages serve as high-conversion entry points for marketing campaigns and organic search traffic. The system supports both manual page creation and automated, AI-driven generation using OpenAI.

## Admin Navigation and Interface

Landing Pages are managed through a dedicated **Landing Pages** menu in the WordPress admin sidebar. The menu contains:

| Menu Item | Description |
| :--- | :--- |
| **All Landing Pages** | View and manage all existing landing pages. |
| **Add New Landing Page** | Create a new landing page manually. |
| **Makes** | Manage the Makes taxonomy used to associate landing pages with vehicle makes. |
| **Models** | Manage the Models taxonomy used to associate landing pages with vehicle models. |
| **Body Types** | Manage the Body Types taxonomy for landing pages. |
| **Locations** | Manage the Locations taxonomy for landing pages. |
| **Features** | Manage the Features taxonomy (used for highlighting up to 3 features per page). |
| **Images** | Configure default images for different vehicle body types. |
| **Generate with AI** | Access the automated AI-powered page generation tool. |

## Creating Landing Pages

### Manual Creation

When creating a landing page manually (**Landing Pages → Add New Landing Page**), the classic editor is used. The edit screen contains the following fields and meta boxes:

**Taxonomy Panels (sidebar):**

| Panel | Type | Description |
| :--- | :--- | :--- |
| **Makes** | Checkbox taxonomy | Associate the page with one or more vehicle makes. |
| **Models** | Checkbox taxonomy | Associate the page with one or more vehicle models. |
| **Body Types** | Tag taxonomy | Associate the page with a body type (e.g., "dealership", "SUV"). |
| **Locations** | Tag taxonomy | Associate the page with a location (e.g., "Nova Scotia"). |
| **Features** | Tag taxonomy | Select up to 3 features to highlight on the page (e.g., "Heated Seats"). |
| **Order** | Numeric field | Controls the display order of the landing page. |

**Generated Content Meta Box:**

A dedicated meta box below the main editor contains structured content fields:

| Field | Description |
| :--- | :--- |
| **Headline** | The main page headline. |
| **Subheadline** | A supporting subheadline beneath the main headline. |
| **Opening Paragraph** | An introductory paragraph for the page. |
| **Before Content Subheading** | A subheading that appears before the main content block (e.g., "Guaranteed Financing – We'll Get You Approved"). |
| **Main Content** | The primary body copy. Supports basic rich text. |
| **CTA Text** | The call-to-action button label (e.g., "Shop All Vehicles in Nova Scotia"). |
| **CTA Link** | The URL the CTA button links to (e.g., `/inventory/`). |
| **Benefits** | A list of bullet-point benefits, one per line (e.g., "116-point mechanical inspection", "New MVI within 30 days"). |

**Permalink:** Landing pages use the `/landing/` prefix by default (e.g., `yourdealership.com/landing/used-dealership-nova-scotia/`).

### AI-Powered Generation

The **Generate with AI** tool (`Landing Pages -> Generate with AI`) allows you to quickly create comprehensive landing pages by providing just a few key parameters.

1.  **Select Target**: Choose the Make, Model, Body Type, and Location for the page.
2.  **Configure AI**: Select the desired tone (e.g., Professional, Enthusiastic) and the OpenAI model to use.
3.  **Generate**: The system uses OpenAI to generate structured content, including:
    -   **Headline & Subheadline**: Compelling titles for the page.
    -   **Opening Statement**: An introductory paragraph tailored to the selection.
    -   **Main Content**: Detailed copy highlighting the benefits of the selected vehicles in that location.
    -   **Benefits List**: A bulleted list of key selling points.
    -   **Call to Action (CTA)**: Persuasive text and a link for the primary conversion goal.

Generated pages are saved as **Drafts**, allowing you to review and refine the content before publishing.

## Dynamic Content Components

Landing pages are designed to be dynamic, automatically pulling in relevant inventory and media based on their configuration.

### Shortcodes

Use these specialized shortcodes within your landing page content or templates:

-   **`[landing_page_image id="1"]`**: Displays one of the default images configured for the page's assigned **Body Type**. You can manage these defaults in `Landing Pages -> Images`.
-   **`[landing_page_benefits]`**: Automatically renders the benefits list (either manually entered or AI-generated) as a styled list with checkmark icons.
-   **`[sdas_vehicle_grid]`**: When used on a landing page, this shortcode automatically filters the inventory to match the page's assigned Make, Model, Body Type, and Location.

### Default Images by Body Type

The **Landing Pages -> Images** screen allows you to set up a library of default images for each vehicle body type (e.g., SUV, Sedan, Truck). 

-   You can upload multiple images per body type.
-   The `[landing_page_image]` shortcode references these by their index (e.g., `id="1"`, `id="2"`).
-   If a specific body type doesn't have an image assigned, the system can fall back to a global "Default Body Type" selection.
