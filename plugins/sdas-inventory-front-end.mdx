## Overview

SDAS Inventory provides a comprehensive suite of shortcodes to display vehicle data and interactive tools on your WordPress site. These shortcodes are designed to work with any theme and are compatible with modern page builders like **Breakdance**, **Beaver Builder**, and **Elementor**.

This guide covers how to wire up the primary shopping journey, configure the Vehicle Detail Page (VDP), and enable personalization features like saved vehicles and comparisons.

## The Visitor Journey

When properly configured, SDAS Inventory supports a complete, modern car shopping experience:

1.  **Discovery**: Visitors land on a main inventory page powered by `[sdas_vehicle_grid]` (often paired with `[sdas_filters]`). They browse, sort, and filter the available stock.
2.  **Exploration**: Clicking a vehicle card takes them to the **Vehicle Detail Page (VDP)**. Here, they see high-resolution galleries, detailed specifications, and payment estimates.
3.  **Personalization**: While browsing, visitors can click "Save" or "Compare" on specific vehicles. These choices are saved to their session.
4.  **Decision**: Visitors navigate to dedicated "Saved Vehicles" or "Compare" pages to review their shortlisted options side-by-side before submitting a lead form.

## Core Browsing Shortcodes

These shortcodes build the primary discovery phase for your visitors.

### `[sdas_vehicle_grid]`
Displays a grid or list of vehicles. This is the primary way to show multiple vehicles on a page. The grid uses a modern, high-conversion card design that automatically adapts to mobile and desktop views.

<Callout kind="info">
**SEO Benefit**: When you place `[sdas_vehicle_grid]` on a standard WordPress page, SDAS automatically detects it and emits `CollectionPage` and `ItemList` Schema.org structured data for the vehicles shown in the grid.
</Callout>

<Callout kind="warning">
**Grid vs. Filters**: If you are using the **SDAS Vehicle Filters** plugin, you should use `[sdas_filter_results]` instead of `[sdas_vehicle_grid]` on your main search page. `[sdas_vehicle_grid]` is best used for standalone displays (e.g., "Featured Vehicles" on the homepage) where dynamic filtering is not required.
</Callout>

**Available Attributes:**

| Attribute | Default | Description |
| :--- | :--- | :--- |
| `per_page` | `12` | Number of vehicles to display per page. |
| `orderby` | `date` | Sort by `date`, `price`, `mileage`, `year`, or `title`. |
| `order` | `DESC` | Sort order (`ASC` or `DESC`). |
| `view` | `grid` | Initial view style: `grid` or `list`. |
| `columns` | `3` | Number of columns in grid view. |
| `enable_view_toggle` | `true` | Whether to show the grid/list view toggle. |
| `pagination_style` | `numbers` | Pagination style: `numbers`, `simple`, or `load_more`. |
| `show_compare_button` | `false` | Whether to show the "Add to Compare" button on cards. |
| `show_quick_view` | `false` | Whether to show the "Quick View" button on cards. |
| `awards_style` | `ribbon_banner` | How to display awards: `ribbon_banner` or `text_badges`. |

**Display Options (Show/Hide Fields):**
You can toggle specific vehicle details on the cards using `true` or `false`:
`show_year`, `show_make`, `show_model`, `show_price`, `show_condition`, `show_body_type`, `show_transmission`, `show_mileage`, `show_stock_number`, `show_vin`, `show_location`, `show_drivetrain`, `show_engine`, `show_fuel_type`, `show_nhtsa_rating`, `show_awards`.

Example:
```text
[sdas_vehicle_grid per_page="24" show_year="true" show_make="true" show_model="true" show_compare_button="true"]
```

### `[sdas_vehicle_search]`
Displays a search bar that allows visitors to find vehicles by keyword, VIN, or stock number.

**Available Attributes:**

| Attribute | Default | Description |
| :--- | :--- | :--- |
| `style` | `hero` | Visual style: `hero`, `compact`, or `inline`. |
| `placeholder` | `Search vehicles...` | Placeholder text for the search input. |
| `show_autocomplete` | `true` | Whether to show live search suggestions. |
| `search_fields` | `vin,stock,make,model,year,description` | Comma-separated list of fields to search. |

Example:
```text
[sdas_vehicle_search style="compact" placeholder="Search by VIN or Model..."]
```

## Vehicle Detail Page (VDP) Setup

The VDP is the single post template for the `sdas_vehicle` custom post type. If you are using a page builder like Breakdance or Elementor, you should create a **Single Post Template** assigned to "Vehicles" and use the following shortcodes to build the layout.

- **`[sdas_vehicle_gallery]`**: Displays the vehicle's photo gallery. It automatically handles the main image and thumbnail slider.
- **`[sdas_vehicle_spec_sheet]`**: Outputs a structured table of vehicle specifications (engine, transmission, dimensions, etc.).
- **`[sdas_payment_calculator]`**: Adds an interactive payment estimator. *Note: This requires global calculator presets to be configured in SDAS Settings -> Calculator.*
- **`[sdas_similar_vehicles]`**: Shows a grid of related vehicles based on segment or price. Useful for keeping visitors engaged if the current vehicle isn't a perfect fit.
- **`[sdas_vdp_cta]`**: Displays a call-to-action block (e.g., "Get a Quote" or "Book a Test Drive").

## Personalization and Comparison

Turn on richer shopping tools by creating dedicated pages for these interactive shortcodes:

- **`[sdas_vehicle_compare]`**: Renders the side-by-side comparison view. **You must place this on a dedicated page** (e.g., `/compare/`) and ensure the page URL is configured in SDAS Settings so the "Compare" buttons know where to link.
- **`[sdas_compare_button]`**: Adds a button to add the current vehicle to the comparison list. (Usually placed on the VDP).
- **`[sdas_save_vehicle_button]`**: Adds a "Save" or "Favorite" button to a vehicle.
- **`[sdas_saved_vehicles]`**: Displays the visitor's list of saved vehicles. Place this on a dedicated `/saved-vehicles/` page.
- **`[sdas_recently_viewed]`**: Shows a list of vehicles the visitor has recently browsed.

## Dynamic Content Shortcodes

These shortcodes are useful for landing pages and SEO-focused content:

- **`[sdas_body_type]`**: Dynamically outputs the body type of the current vehicle or landing page context.
- **`[sdas_city_name]`**: Outputs the dealership's city name for local SEO.
- **`[landing_page_image]`**: Displays the default image for a specific body type (configured in SDAS Inventory settings).
- **`[landing_page_benefits]`**: Outputs a list of dealership benefits (e.g., "The Super Dave's Advantage").

## The Vehicle Edit Screen

When you create or edit a vehicle in **Vehicles → All Vehicles → Edit**, the edit screen contains the following meta boxes and taxonomy panels:

**Taxonomy Panels (sidebar):**

| Panel | Type | Description |
| :--- | :--- | :--- |
| **Makes** | Checkbox taxonomy | The vehicle manufacturer (e.g., Hyundai, Toyota). |
| **Models** | Checkbox taxonomy | The vehicle model (e.g., ELANTRA, Civic). |
| **Years** | Tag taxonomy | The model year (e.g., 2017). |
| **Body Types** | Tag taxonomy | The body style (e.g., Car, SUV, Truck). |
| **Conditions** | Tag taxonomy | The vehicle condition (e.g., Super Certified, Used, New). |
| **Transmissions** | Tag taxonomy | The transmission type (e.g., Automatic, Manual). |
| **Statuses** | Tag taxonomy | The availability status (e.g., Available, Sold, In Transit). |
| **Locations** | Tag taxonomy | The dealership location or lot. |
| **Colours** | Tag taxonomy | The exterior colour (e.g., Black, White). |

**Meta Boxes:**

| Meta Box | Description |
| :--- | :--- |
| **AI Description Generator** | Generates a compelling vehicle description using AI based on the vehicle's features. If a description already exists, generating a new one will replace it. |
| **Vehicle Details** | Core vehicle data fields: VIN, Price, Stock Number, Mileage, Number of Seats, XML Import ID, Trim, Drivetrain, Transmission Description, Engine Description, Engine Cylinders, Engine Displacement, Exterior Color, Interior Color, Fuel Type, City MPG, Highway MPG, and Vehicle Options. |
| **Vehicle Images** | Manages the vehicle photo gallery. Shows a count of Downloaded vs. External images and the total. The featured image is marked with a star. |

The vehicle's **title** (set in the main content area) typically follows the format `YEAR MAKE MODEL TRIM` (e.g., "2017 Hyundai ELANTRA GL"). The **content area** holds the vehicle description, which can be written manually or generated by the AI Description Generator.

## Setup & QA Workflow

Follow this checklist to ensure your inventory front-end is fully operational.

<Steps>

<Step title="Check prerequisites" icon="check-circle">
Ensure SDAS Core and SDAS Inventory are active. Go to **Settings -> Permalinks** and click "Save Changes" to ensure the `/vehicle/` routes are registered.
</Step>

<Step title="Create your main Inventory page" icon="plus">
Create a new page (e.g., "Inventory"). If using filters, use `[sdas_filters]` and `[sdas_filter_results]`. If not, use `[sdas_vehicle_grid]`.
</Step>

<Step title="Build the VDP Template" icon="layout">
Using your theme or page builder, create a template for single `sdas_vehicle` posts. Insert the gallery, spec sheet, and calculator shortcodes.
</Step>

<Step title="Configure Inventory Settings" icon="settings">
Navigate to **Vehicles → Inventory Settings** to configure two key areas:

**General Settings:**
- **Fallback Image URL** — The image displayed whenever a vehicle has no photos (featured, stock, or gallery). Provide a full URL to an image on your site or CDN.

**Export Cache Settings:**
- **Enable Export Caching** — Caches the vehicles export endpoint for improved performance. Recommended for large inventories.
- **Cache Refresh Frequency** — How many times per day the cache refreshes (e.g., 4 times = every 6 hours). Recommended: 3–4 times per day.
- **Cache Status** — Shows the last update time, next scheduled refresh, and number of vehicles currently cached. A **Clear Cache Now** button is available if you need to force an immediate refresh.

Once saved, click **Save Changes** to apply.
</Step>

<Step title="Export vehicle links" icon="download">
On the **Vehicles → All Vehicles** screen, two export buttons appear above the vehicle list:

- **Export Links List** — Downloads a JSON file containing each vehicle’s `stock_number`, `vin`, and `permalink`.
- **Export Links CSV** — Downloads the same data as a CSV file with columns `Stock#`, `VIN`, and `URL`, ready to open in Excel or import into third-party tools.

Both exports respect the active **Status** filter, so you can export only available vehicles if needed. The JSON export is cached according to your Export Cache Settings; the CSV uses the same cached dataset.

<Callout kind="info">
The export payload contains `stock_number`, `vin`, and `permalink` only. The vehicle description field was removed from the export in a recent update to keep the file compact.
</Callout>
</Step>

<Step title="QA the Visitor Journey" icon="monitor">
1. Open an incognito window and navigate to your inventory page.
2. Verify the grid loads and pagination works.
3. Click a vehicle to view the VDP. Verify the gallery loads and the calculator functions.
4. Click "Save" and "Compare" on two different vehicles.
5. Navigate to your Saved and Compare pages to verify the vehicles appear correctly.
</Step>

</Steps>
